# Acceptance Criteria - Habit Tracker MVP

## Cadastro do rastreamento
- o usuário consegue selecionar pelo menos um hábito
- o usuário consegue definir a duração do período
- o rastreamento é salvo localmente
- ao concluir, o usuário entra no fluxo principal do app

## Registro diário
- o usuário consegue criar um novo registro
- o usuário consegue escolher um hábito ativo
- o usuário consegue escolher mood
- o usuário consegue escolher intensidade
- o usuário consegue adicionar uma observação opcional
- o registro salvo aparece no histórico

## Edição
- o usuário consegue editar um registro existente
- os dados atualizados aparecem corretamente após salvar

## Exclusão
- o usuário consegue excluir um registro
- o registro removido deixa de aparecer na lista
- as estatísticas são recalculadas após exclusão

## Estatísticas
- a tela mostra quantidade total de registros
- a tela mostra frequência por hábito
- a tela mostra média de intensidade por hábito
- a tela mostra distribuição de moods
- os dados refletem exatamente o que está salvo

## Configurações
- o usuário consegue adicionar ou remover hábitos do acompanhamento
- o usuário consegue alterar a duração do período
- as alterações ficam persistidas localmente

## Qualidade técnica
- app compila sem erro
- navegação funciona
- tipagem principal está definida
- serviços de persistência estão separados da UI
